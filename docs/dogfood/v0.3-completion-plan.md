# v0.3 Completion Plan

## Goal

Complete v0.3 by running three real feature PRs through the Agent Memory protocol using fresh-context sub-agent review.

## PR 1: Router Versioning + Stale Router Detection

Purpose:
Make installed instructions auditable by version so stale AGENTS.md router blocks can be detected.

Expected feature:

- add router version constant, for example `v0.3`
- include version in managed AGENTS.md block
- update doctor to report router version or stale/missing status
- install-instructions should upgrade stale managed blocks
- tests for installed/current/stale/missing router states

Dogfood question:
Does the installed router reliably tell agents to invoke the right protocol?

## PR 2: Structured Pack JSON + Source Display

Purpose:
Improve the pack output enough for agents and reviewers to understand why memory was included.

Expected feature:

- improve `pack --json` structured output if currently too markdown-only
- include matched memory IDs, type, source, confidence/severity where available
- keep markdown output backward compatible
- avoid retrieval redesign
- tests for JSON structure and source display

Dogfood question:
Was the returned memory pack actually useful before planning?

## PR 3: Dogfood Aggregate / v0.3 Completion Report

Purpose:
Summarize multiple dogfood sessions so v0.3 completion can be reviewed.

Expected feature:

- add a derived command or documented report flow for multiple sessions
- preferred command if scoped safely:
  `agentmem dogfood summarize --sessions ses_a,ses_b,ses_c --json`
- no persistence required
- output aggregate protocol pass count, memory-used count, evidence count, candidate count, friction notes if available
- README/docs update
- tests for aggregation

Dogfood question:
Do three real dogfood sessions provide enough evidence to decide whether v0.3 is usable, useful, and low-noise?

## Completion criteria

v0.3 is complete when:

- three non-trivial dogfood PRs are merged
- each PR has a protocol check result
- each PR has a dogfood report result
- at least one PR shows memory affected behavior or planning
- at least one PR proposes useful reusable learning
- no PR reports severe protocol friction
- docs clearly state what was learned
