# v0.2 Roadmap Archive

This document is retained as historical context. The active release source of truth is [../v1-release/V1_RELEASE_PLAN.md](../v1-release/V1_RELEASE_PLAN.md).

The v0.2 direction was to add a protocol spine around the existing local CLI:

- managed repo instructions
- session start, finish, and receipt
- memory packet loading
- command preflight receipts
- evidence events
- candidate proposal and review
- non-interactive manage planning

Much of that spine exists in the current branch. V1 work builds on it with stronger retrieval metadata, packet structure, update/archive commands, release artifacts, deterministic evaluation, and release verification.

Deferred ideas from the roadmap remain deferred unless explicitly pulled into a release plan:

- embeddings or vector search
- graph memory
- MCP server
- cloud sync
- hosted dashboard
- cross-repo global memory
- automatic long-term memory writes without review
- hard command blocking
